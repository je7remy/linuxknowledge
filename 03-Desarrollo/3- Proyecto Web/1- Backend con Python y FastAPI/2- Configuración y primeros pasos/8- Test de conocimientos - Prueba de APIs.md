
### 1. ¿Qué tipo de peticiones se pueden lanzar desde la barra del navegador?

- A. POST
    
- **B. GET** (Correcta)
    
- C. OPTIONS
    

> Justificación:
> 
> La barra de direcciones del navegador siempre realiza peticiones GET para recuperar recursos (páginas web, imágenes, etc.). No permite enviar métodos como POST o PUT nativamente sin formularios o scripts.

---

### 2. ¿Qué permite hacer una colección en Postman o Thunder Client?

- A. Editar código de la API
    
- **B. Guardar y compartir peticiones** (Correcta)
    
- C. Crear documentación automática
    

> Justificación:
> 
> Las colecciones sirven para agrupar, organizar, guardar configuraciones y compartir un conjunto de peticiones relacionadas con un proyecto o equipo.

---

### 3. ¿Qué permite visualizar la pestaña “Headers”?

- **A. Cabeceras HTTP** (Correcta)
    
- B. Cuerpo de la respuesta
    
- C. Cookies almacenadas
    

> Justificación:
> 
> Permite ver y editar los metadatos de la comunicación HTTP (como el Content-Type, tokens de autorización, User-Agent, etc.) que se envían o reciben.

---

### 4. ¿Para qué se utiliza Postman principalmente?

- **A. Interactuar con APIs** (Correcta)
    
- B. Editar código fuente
    
- C. Diseñar interfaces gráficas
    

> Justificación:
> 
> Es la herramienta estándar para desarrollo y pruebas, permitiendo enviar peticiones HTTP a una API y analizar sus respuestas sin necesidad de programar un frontend.

---

### 5. ¿Qué ventaja ofrece Thunder Client respecto a Postman?

- A. Tiene más protocolos que Postman
    
- **B. Está integrado en VSCode** (Correcta)
    
- C. Es exclusivo para sistemas Linux
    

> Justificación:
> 
> Al ser una extensión, permite realizar pruebas sin cambiar de ventana, manteniendo el código y el cliente API en el mismo entorno de trabajo.

---

### 6. ¿Qué función tiene el botón “Send” en Postman y Thunder Client?

- A. Guardar la colección
    
- **B. Ejecutar la petición** (Correcta)
    
- C. Limpiar los campos
    

> Justificación:
> 
> Es el disparador que construye la petición HTTP con los parámetros configurados y la envía al servidor, esperando la respuesta.

---

### 7. ¿Qué permite hacer Thunder Client sin salir de VSCode?

- **A. Ejecutar peticiones y guardar colecciones** (Correcta)
    
- B. Compilar código
    
- C. Crear bases de datos
    

> Justificación:
> 
> Permite todo el ciclo de prueba básico (lanzar peticiones y organizar historial) directamente desde el editor.

---

### 8. ¿Cuál es una diferencia clave entre Postman y Thunder Client?

- A. Thunder Client tiene más funciones que Postman
    
- B. Ambos son extensiones
    
- **C. Postman es una aplicación independiente, Thunder Client es una extensión para VS Code** (Correcta)
    

> Justificación:
> 
> Postman es un software "standalone" (independiente) más pesado y completo, mientras que Thunder Client es un plugin ligero integrado en el editor.

---

### 9. ¿Qué método HTTP es el más común en navegadores?

- **A. GET** (Correcta)
    
- B. POST
    
- C. DELETE
    

> Justificación:
> 
> Es el método por defecto para la navegación web (traer información).

### 10. ¿Qué información adicional ofrece Postman al realizar una petición?

- A. Solo la URL y el método
    
- B. Solo el código de estado
    
- **C. Tiempo de respuesta, tamaño y cookies** (Correcta)
    

> Justificación:
> 
> Proporciona métricas técnicas detalladas (latencia en ms, tamaño en bytes, cookies de sesión) vitales para depurar y optimizar el rendimiento de la API.
